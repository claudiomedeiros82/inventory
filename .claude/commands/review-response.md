Draft a professional App Store or Play Store review response.

The developer will provide the review text in $ARGUMENTS or paste it after running this command.
If not provided, ask: "Paste the review you'd like to respond to."

Read the review and:
1. Identify the review type: bug report, feature request, confusion, praise, troll/spam
2. Identify the sentiment: 1-2 stars (angry), 3 stars (neutral), 4-5 stars (positive)
3. Identify if there's an actionable issue

Draft a response that:
- Starts with acknowledgment (not defensive)
- Is specific to their issue (not a generic template)
- Gives a concrete next step if there's an action to take
- Is under 350 characters (App Store visible limit)
- Never argues with the user, even if they're wrong
- Never promises features you haven't built

**Response templates by type:**

Bug report:
"Thanks for reporting this — [specific issue] is something we're looking into.
If you're still experiencing it, [specific action: email/update/step].
We'll have a fix in the next update."

Feature request:
"Really appreciate the suggestion! [Feature] is something we'd love to add.
We've noted it for a future update. In the meantime, [workaround if any]."

Confusion / UX issue:
"Thanks for the feedback! [Feature] works by [brief explanation].
If you're still stuck, reach out at [support email] and we'll help you out."

Positive review:
"Thank you! Really glad [specific thing they mentioned] is working well for you.
More updates coming soon."

After drafting, note:
- Is this issue worth fixing? (if yes, suggest creating a task for it)
- Is this feedback pattern appearing in multiple reviews? (flag for product team)
